---Requerimientos funcionales---

Ticket de entrada:
-Registar número de asiento con dependencia a fila.

Gestión de pago de cuota:
-Registrar fecha de pago de cuota.
-Consultar fecha de pago de cuota.
-Registrar al empleado responsable de la gestión de pago de cuota.
-Consultar monto de pago de cuota.
-Registrar estado del pago de cuota.
-Modificar estado del pago de cuota.
-Consultar estado el pago de cuota.
-Generar recibo por cancelación de cuota/s.

Cuota:
-Registrar número de cuota.
-Consultar fecha de alta de la cuota.
-Registrar monto de la cuota.
-Consultar monto de la cuota.

Estado de pago:
-Registrar nombre del estado de pago.
-Registrar fecha de cambio del estado de pago.
-Consultar fecha de cambio del estado de pago.
-Consultar estado del estado de pago.
-Modificar estado del estado de pago.


Código diagrama de clases en PlantUML:

GestionPagoCuota "1" --> "1" EstadoPago
GestionPagoCuota "1" --> "1" Empleado
GestionPagoCuota "1" --> "1..*" Cuota
TicketEntrada  "1" --> "1" Fila


class TicketEntrada {
    -numeroAsiento: Int
    -fila: Fila
    +getFila(): Fila
}

class GestionPagoCuota {
    -cuota: Cuota
    -fechaPago: DateTime
    -responsable: Empleado
    -monto: Int
    -estadoPago: EstadoPago
  
    +getEmpleado(): Empleado
    +getCuota(): Cuota
    +estadoPago(): EstadoPago
    +pagarCuota(): void
    +emitirRecibo() : void
}

class Cuota {
    -numeroCuota: Int
    -fechaCuota: DateTime
    -monto: Int
}

class EstadoPago {
     -fechaAlta: DateTime
     -fechaCambioEstado: DateTime
     -nombre: String
     -estado: Bool
}
